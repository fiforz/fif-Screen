# FifScreen 0.5.2

本版本优先修复 0.5.1 的 USB 回归，并提高局域网连接在不同 Android 设备和路由器上的兼容性。

## 主要修复

- 控制中心只会启动与自身版本一致的 Windows Host 和软件设备 Launcher；找不到匹配版本时会停止启动，不再继续拉起 Android 后误报控制通道关闭。
- Windows 选择 USB 调试时会明确要求 Android 切回 USB，不再受手机上一次无线连接设置影响。
- Windows 分别建立并校验控制端口 27183 和视频端口 27184 的 ADB reverse。
- Host 在扩展显示尚未完成枚举时最多等待 15 秒，避免过早关闭视频通道造成 `socket closed`。
- Android 将底层 socket 错误转换为按连接阶段区分的中文提示，并持续自动重连。

## 局域网改进

- 局域网 UDP 发现和 TCP 连接绑定到 Wi-Fi 或以太网，避免被移动数据、VPN 或厂商网络栈错误路由。
- Android 支持手动输入电脑 IPv4，可绕过禁止广播或启用客户端隔离的网络。
- Windows 控制中心显示本机 IPv4，并增加“应用 PIN 并等待连接”操作。

## 验证结果

- 真实 Android 设备 USB 控制握手和视频通道通过，1920 x 1080 输出稳定约 49–50 FPS。
- Android 单元测试、APK 构建和 Lint 通过。
- Windows 协议、PIN 加密和触控注入测试通过。
- 0.5.2 开发预览安装包构建和负载版本校验通过。

> 当前安装包仍为测试签名驱动的开发预览版。目标 Windows 必须允许测试签名驱动，安装程序不会修改 BCD、安全启动或自动重启系统。
