# FifScreen 0.5.5

本版本修复部分 Windows 10 电脑上的 Android 扩展屏能够正常操作、但始终看不到鼠标的问题。

## 主要修复

- Windows Host 将高对比度软件箭头直接写入待编码的视频帧，不再依赖不同电脑表现不一致的系统光标资源和 `DrawIconEx` 合成结果。
- Host 声明 Per-Monitor V2 DPI 感知，并使用物理坐标判断鼠标是否位于 FifScreen 扩展屏。
- FifScreen 位于主屏左侧或上方且 Windows 物理光标接口返回访问拒绝时，Host 可将有效的屏内局部坐标安全换算为全局负坐标。
- 增加结构化光标状态日志，以及负坐标换算和软件箭头像素回归测试。

## 行为说明

- 扩展屏视频中的鼠标统一显示为固定的黑白高对比度箭头，不随 Windows 的文本、缩放或忙碌光标形状变化。
- 本机系统光标仍由 Windows 自己显示；视频捕获不会使用 `CAPTUREBLT`，不会重新引入主屏光标闪烁。

## 验证结果

- 在两台 Windows 10 电脑上完成真实 USB、Android 视频与鼠标验证；主屏光标不闪烁，左侧负坐标扩展屏能够显示鼠标。
- Windows Release 构建及协议、控制中心、加密、触控、Socket、光标坐标和软件光标测试全部通过。
- Android 单元测试、Lint 和 Debug APK 构建通过。
- 显示驱动通过 Signability 与测试签名验证，Development 安装包完成构建验证。

> 当前安装包仍为测试签名驱动的开发预览版。目标 Windows 必须允许测试签名驱动，安装程序不会修改 BCD、安全启动或自动重启系统。
