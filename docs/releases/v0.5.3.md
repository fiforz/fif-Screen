# FifScreen 0.5.3

本版本修复虚拟显示器被 Windows 放入复制拓扑、GUI 连接模式丢失，以及源码控制中心误选旧二进制导致 USB 驱动误报异常的问题。

## 主要修复

- 控制中心会等待 FifScreen 驱动和 Monitor 就绪，主动切换并验证独立扩展桌面后才启动 Host 和 Android。
- Host 通过 DisplayConfig adapter path 定位 FifScreen 捕获源，不依赖本地化显示名称，复制拓扑下也不会误选物理主屏。
- 修复选择无线局域网并应用 PIN 后，Host 仍以 USB 模式启动的问题。
- 源码控制中心在多个同版本构建并存时选择最新产物；旧 Launcher 状态协议会被明确拒绝，不再误报驱动异常。
- 控制中心和 Host 增加连接模式、进程路径、启动参数及最终 transport 诊断，不记录用户 PIN。

## 验证结果

- Windows Release 构建以及协议、PIN 加密、控制中心回归和触控注入测试全部通过。
- Android 单元测试、Lint 和 Debug APK 构建通过。
- 显示驱动重新构建，通过 Signability 和测试签名验证。
- 在真实设备上验证独立扩展桌面、USB 双 ADB reverse、Android 控制/视频握手和触控。
- 0.5.3 Development 安装包完成构建、负载版本和安装验证。

> 当前安装包仍为测试签名驱动的开发预览版。目标 Windows 必须允许测试签名驱动，安装程序不会修改 BCD、安全启动或自动重启系统。
