# 更新日志

本项目按语义化版本记录用户可见的变化。

## 0.5.2 - 2026-07-12

### 修复

- 修复部分电脑上软件显示设备已经创建、但 Windows 尚未完成显示枚举时，主机立即关闭 USB 视频通道并导致 Android 只提示 `socket closed` 的问题；
- 修复控制中心版本和 Android 版本一致、但实际选中了旧版 Host 时仍继续启动 Android，最终被误报为“电脑关闭了控制通道”的问题；
- Windows 选择 USB 或无线模式时现在会把连接方式明确传给 Android，避免 Android 沿用上次无线偏好而阻断 USB；
- Windows 主机现在等待扩展显示真正可捕获后再开始视频流，并为显示等待开始、成功和超时输出明确诊断事件；
- Windows 控制中心分别配置并校验 USB 控制与视频端口的 ADB reverse，映射失败时不再错误地记录“已配置”；
- Android 按发现、PIN、控制握手、视频认证、视频启动和传输阶段显示中文错误，正常断线继续自动重连，不再向用户暴露底层 socket 文本；
- 修复拔掉 USB 后局域网发现可能被移动数据、VPN 或厂商网络栈路由到错误接口的问题。

### 改进

- Android 局域网发现和后续 TCP 连接会绑定到 Wi-Fi 或以太网，并保留默认网络回退；
- Android 增加手动电脑 IPv4 连接方式，可绕过不转发广播的路由器或 Android 设备；
- Windows 控制中心增加“应用 PIN 并等待连接”按钮，并显示可在 Android 手动输入的本机 IPv4 地址；
- 源码运行时严格选择与当前版本一致的 Host 和 Launcher，找不到匹配产物会在启动 Android 前明确失败；
- Android 16 无线连接按系统要求申请附近设备权限，USB 模式不受影响。

### 验证

- 在真实 Android 设备上验证双 ADB reverse、USB 控制握手、1920 x 1080 约 49–50 FPS 视频、无扩展屏时 15 秒等待、中文错误提示和持续自动重连；
- 通过 Android 单元测试、APK 构建、Lint、MSVC Windows 主机构建及协议和 PIN 加密测试。

## 0.5.0 - 2026-07-11

### 新增

- Windows 控制中心和 Android 应用支持手动选择 USB 调试或无线局域网连接；
- 同一局域网内通过 UDP 自动发现电脑，Windows 与 Android 输入相同四位 PIN 即可握手连接；
- 无线控制通道使用 PBKDF2-HMAC-SHA256 挑战响应和双向证明，视频通道使用一次性会话证明；
- 预留 Android 服务器端点目录、Windows 中继传输枚举和服务器 URL 配置接口，便于后续增加互联网中继。

### 改进

- 无线模式复用现有 1920 x 1080、目标 50 FPS H.264 和原生触控链路；
- Windows 防火墙规则只允许本地子网访问 UDP 发现和 TCP 传输端口，完整卸载时自动清理；
- Android 连接设置在启动、断线或按返回键时可访问，PIN 仅保存在进程内存中；
- USB 模式继续只监听 `127.0.0.1` 并使用原有 ADB reverse，不增加无线配对开销。

### 验证

- 在真实 Android 设备与 Windows 电脑直连局域网中验证 PIN 接受/拒绝、1920 x 1080 约 50 FPS 视频和触控；
- 验证从无线模式切回 USB 后，原有画面、触控、Owner 和 Software Device 均保持正常。

## 0.4.1 - 2026-07-11

### 修复

- 修复源码控制中心可能启动无触控能力的旧版 Windows 主机，导致手机触摸无响应的问题；
- 控制中心现在校验主机和设备启动器的文件版本，并优先使用与当前版本一致的构建产物；
- 检测到旧主机时只重启主机进程，保留现有扩展显示 Owner 和 Software Device，避免重复创建设备；
- Android 触摸视图现在始终接管完整手势，不再因发送线程瞬时未就绪而丢失后续移动或抬起事件。

### 改进

- 增加触摸握手能力、输入就绪状态和手势发送结果的结构化诊断日志；
- 在真实 Android 设备、1920 x 1080 FifScreen 扩展屏上验证 `WM_POINTER` 和鼠标兼容消息。

## 0.4.0 - 2026-07-11

### 新增

- Android 扩展屏支持原生单指和多指触控；
- 支持点击、拖动、长按以及由 Windows 应用处理的双指缩放和滚动手势；
- Windows 主机使用系统 `InjectTouchInput` 将触点注入 FifScreen 扩展显示器；
- 新增触控协议、坐标映射、单指、多指和断线取消测试。

### 改进

- Android 使用独立低延迟输入发送线程，并合并连续移动帧，避免触控积压；
- 握手增加触控能力协商，新旧 Android 和 Windows 组件可以兼容混用；
- 控制连接断开时自动取消所有活动触点，避免残留按下状态。

## 0.3.2 - 2026-07-10

### 修复

- 修复 GitHub Release 静态清单以二进制响应返回时，Windows PowerShell 5.1 无法解析 JSON、错误提示连接失败的问题；
- 静态清单现在先按 UTF-8 显式解码，再进行字段和 SHA-256 校验；
- 更新检查显式启用 TLS 1.2，并关闭 PowerShell 下载进度渲染以减少网络请求阻塞。

## 0.3.1 - 2026-07-10

### 新增

- Windows 控制中心启动时检查管理员权限；
- 非管理员启动时提示“请以管理员身份运行”，确认后立即退出；
- GitHub Release API 不可用时回退到最新 Release 的静态更新清单资产。

### 修复

- 修复网络正常但 GitHub API 间歇性超时导致手动检查立即报错的问题；
- 手动检查更新增加三次重试，只有 API 和静态清单都失败才显示网络提示；
- 后台检查继续使用静默退避重试，不打扰用户。
